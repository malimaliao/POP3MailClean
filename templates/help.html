{{MC_name}}《版本：{{MC_version}}》

本系统采用B/S架构，基于Python3 + Flask + Sqlite3环境部署，使用poplib、smtplib、SocketIO 、HTML5实现远程邮件账号的批量清理和实时任务跟踪。

Python环境及依赖模块：
1，Python 3.7 +
2，Flask 1.1.1
3，flask-socketio 4.3.0
4，SQLite 3

1，本系统使用方法

本软件运行后，会自动建立一个HTTP网站。如果在本机使用，则通过Chrome浏览器（或其他支持HTML5的浏览器）访问网址：http://本机IP地址:8088
其中，也可以在其他电脑通过IP地址访问这个运行软件，而8088是默认的系统访问端口号。

2，邮件服务器配置

通过notepad++ 或其他编码工具修改本系统目录下的start.py：

MC_system_user = 系统后台账号
MC_system_pass = 系统后台密码
MC_web_ip = 本软件运行时使用IP地址（文本型，一般不建议修改），即当本软件处于多IP环境下，可以指定运行在某个IP上
MC_web_port = 本软件运行时使用的web端口号（整数型，一般不建议修改），当端口冲突时候可以修改此端口解决冲突问题
MC_pop3_host = 要管理的邮件服务器地址（文本型）
MC_pop3_port = 要管理的邮件服务器端口（整数型）
MC_report_task = 是否每次任务结束时发送任务报告（逻辑型）
MC_report_send_login = 任务报告使用的邮箱账号，报告启用则必填
MC_report_send_password = 任务报告使用的邮箱账号密码，报告启用则必填
MC_report_send = 使用的邮件发送账号（文本型），报告启用则必填
MC_report_to = ['xxx@qq.com'] # 接收邮箱地址，允许多个，格式：['xxx@163.com', 'xxx@qq.com']，报告启用则必填
MC_report_cc = ['xxx@tet.com'] # 抄送邮箱，，允许多个，格式：['xxx@163.com', 'xxx@qq.com']，不需要则注释掉或者为[]

3，邮箱账号的管理

可以通过SQLiteStudio这个数据库管理软件直接连接本系统目录下的mail_clean.db，进行数据的备份管理。

4，关于清理账号

清理账号即需要清理的邮箱服务器的邮箱账号（部分邮箱账号登录是不需要加@domain.com），而保留天数是服务器上保留的邮件存留时间，例如保留3天，则自今日起算。
